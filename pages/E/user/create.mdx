## Create User Subscription

### HTTP Request

**POST** `http://localhost:3000/v1/user_subscription/create`

### Authentication

This endpoint requires a Bearer Token for authentication.

```plaintext
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJwaG9uZSI6Iis5MTk4MTI5NDA3MDYiLCJ1aWQiOiIwXzZpaGFjanJyIn0.Rf1ZUJpJdBBbep4cGCqwBQ2A8IyETp2UTrnecGHdi6I
```

### Request Body

The request body should be in JSON format.

```json
{
  "subscription_id": "ftx0li8efd",
  "start_date": "27/11/2004",
  "end_date": "27/11/2004"
}
```

### Response

A successful response will return a status code of `200 OK` along with the created user subscription details in JSON format.

### Example

**Request:**

```http
POST /v1/user_subscription/create HTTP/1.1
Host: localhost:3000
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJwaG9uZSI6Iis5MTk4MTI5NDA3MDYiLCJ1aWQiOiIwXzZpaGFjanJyIn0.Rf1ZUJpJdBBbep4cGCqwBQ2A8IyETp2UTrnecGHdi6I
Content-Type: application/json

{
  "subscription_id": "ftx0li8efd",
  "start_date": "27/11/2004",
  "end_date": "27/11/2004"
}
```

**Response:**

```json
{
  "subscription_id": "ftx0li8efd",
  "start_date": "27/11/2004",
  "end_date": "27/11/2004",
  "user_id": "0_zihazjr",
  "status": "active"
}
```

### Notes

- Ensure that the Bearer Token is valid and included in the request header.
- Both `start_date` and `end_date` should be in `dd/mm/yyyy` format.
- The `subscription_id` must be a unique identifier for the subscription.
