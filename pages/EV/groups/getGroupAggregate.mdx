## Overview

This document describes the `getGroupAggregate` API endpoint, which is used to retrieve aggregate data for a specific group. This endpoint requires authentication via a Bearer token.

## HTTP Request

### GET

`GET http://localhost:3000/v1/groups/2/aggregate`

- **Authorization:** Bearer token

## Authentication

To access this API endpoint, you need to include an authorization header with a valid Bearer token. The format of the header is:

```
Authorization: Bearer <token>
```

## Request Parameters

- **URL Parameters:** None
- **Request Body:** None

## Response

On a successful request, the server will return a JSON response containing the aggregate data for the specified group. The exact structure of the response will depend on the data available for the group.

## Error Handling

In case of an error, the server will return an appropriate error message and HTTP status code. Possible errors include:

- **401 Unauthorized:** If the provided token is missing or invalid.
- **404 Not Found:** If the group ID does not exist.
- **500 Internal Server Error:** If there is a problem on the server side.

## Conclusion

The `getGroupAggregate` API endpoint is a secure way to retrieve aggregated metrics for a particular group. Proper authentication with a Bearer token is crucial to access this resource.
