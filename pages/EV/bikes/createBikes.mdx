## Overview

The `createBikes` API is used to create multiple bike entries in the system. This document provides a detailed description of the API endpoint, including the request and response details.

## Endpoint

### POST
`http://localhost:3000/v1/bikes/create`

## Authentication

This endpoint does not require authentication. However, it includes a custom header for admin authorization.

### Headers

| Key                      | Value                                                                                 |
|--------------------------|---------------------------------------------------------------------------------------|
| X-Karma-Admin-Auth       | CoffeeCodesMelloBxAparnaEewqjbdkwqbkdjoehui3r7328grehjvdjvchjiuiudhsjkhkjsadab         |

## Request Body

The request body should be in JSON format and contain an array of bike objects. Each bike object should have the following fields:

- `frame_number`: String (Unique identifier for the bike's frame)
- `name`: String (Name of the bike)
- `station_id`: String (ID of the station where the bike is located)
- `top_speed`: Number (Top speed of the bike in km/h)
- `range`: Number (Range of the bike in km)
- `time_to_station`: Number (Time in minutes to reach the station)
- `images`: Array of Strings (URLs of bike images)

## Response

Upon successful creation of the bike entries, the API will return a success message and details of the created bikes.

## Error Handling

In case of errors, the API will return an appropriate error message with a corresponding HTTP status code.

### Example Errors

- **400 Bad Request**: Validations errors or missing required fields.
- **500 Internal Server Error**: Something went wrong on the server side.
