## Overview

The `GetUserTripsByFastestSpeed` API endpoint provides a method to retrieve user trips sorted by the fastest speed. This documentation outlines the necessary details to successfully use this endpoint.

## HTTP Request

- **Method:** GET
- **URL:** `http://localhost:3000/v1/trips/speed/fastest`

## Query Parameters

| Parameter | Type | Description           | Default Value |
| --------- | ---- | --------------------- | ------------- |
| `limit`   | int  | Number of trips to retrieve | 10            |

## Authentication

This endpoint requires Bearer token authentication. Pass the token in the `Authorization` header in the following format:

```
Authorization: Bearer <TOKEN>
```

## Response

- **Status Code:** 200 OK
- **Content:** JSON array of trips sorted by speed

```json
[
  {
    "trip_id": "<trip_id>",
    "user_id": "<user_id>",
    "speed": "<fastest_speed>",
    "other_details": "<other_trip_details>"
  },
  ...
]
```

## Notes

The `limit` query parameter is optional and defaults to `10`. Use this parameter to control the number of trip records returned by the API. This endpoint only supports queries that return the top fastest trips.
