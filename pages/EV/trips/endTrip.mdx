# endTrip API

This document provides the necessary details to interact with the endTrip API endpoint, which marks the end of a trip.

## API Information

- **Name:** endTrip
- **Type:** HTTP
- **Sequence:** 2

## Endpoint Details

- **Method:** POST
- **URL:** `http://localhost:3000/v1/trips/end`
- **Body Format:** JSON
- **Authentication:** Bearer Token

## Request Body

The request body must be in JSON format with the following structure:

```json
{
  "id": "ulj1iq0-0w",
  "bike_id": "i6bjwk0yhu",
  "station_id": "a2m392c58f",
  "start_timestamp": "2024-09-02T07:15:00Z",
  "end_timestamp": "2024-09-02T07:50:00Z",
  "distance": 15.8,
  "duration": 35.0,
  "average_speed": 27.1,
  "path": "base64_encoded_path_data_3"
}
```

### Request Body Parameters

- **id** (string): The unique identifier for the trip.
- **bike_id** (string): The unique identifier for the bike used in the trip.
- **station_id** (string): The unique identifier for the station where the trip ends.
- **start_timestamp** (string): The ISO 8601 date-time string indicating when the trip started.
- **end_timestamp** (string): The ISO 8601 date-time string indicating when the trip ended.
- **distance** (number): The total distance traveled during the trip in kilometers.
- **duration** (number): The duration of the trip in minutes.
- **average_speed** (number): The average speed during the trip in kilometers per hour.
- **path** (string): The base64-encoded path data of the trip.

## Summary

This endpoint allows you to mark the end of a trip by providing all relevant trip details. Ensure that the bearer token is properly included in the request for authentication, and that the body is structured as specified.
