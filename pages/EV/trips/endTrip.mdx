## Meta Information

This section provides basic meta information about the `endTrip` API:

- **Name:** endTrip
- **Type:** HTTP
- **Sequence:** 2

## Endpoint

The `endTrip` API endpoint is used to mark the end of a trip. The details are as follows:

- **Method:** POST
- **URL:** `http://localhost:3000/v1/trips/end`
- **Body Type:** JSON
- **Authentication:** Bearer Token

## Request Body

The request body must be in JSON format and contain the following fields:

- **id** (string): Unique identifier for the trip
- **bike_id** (string): Identifier for the bike used in the trip
- **station_id** (string): Identifier for the station where the trip ends
- **start_timestamp** (string, ISO 8601): Timestamp when the trip started
- **end_timestamp** (string, ISO 8601): Timestamp when the trip ended
- **distance** (number): Distance covered in the trip (in kilometers)
- **duration** (number): Duration of the trip (in minutes)
- **average_speed** (number): Average speed during the trip (in km/h)
- **path** (string): Base64 encoded path data representing the trip route

Example of a valid request body:

```json
{
  "id": "ulj1iq0-0w",
  "bike_id": "i6bjwk0yhu",
  "station_id": "a2m392c58f",

  "end_timestamp": "2024-09-02T07:50:00Z",
  "distance": 15.8,
  "duration": 35.0,
  "average_speed": 27.1,
  "path": "base64_encoded_path_data_3"
}
```
