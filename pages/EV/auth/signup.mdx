# Signup API

## Overview

The Signup API allows users to create a new account in the system. It accepts user details and credentials, and upon successful account creation, the user is registered in the system.

### Endpoint Information

- **Type**: HTTP
- **Sequence**: 3

## HTTP Request

- **Method**: POST
- **URL**: `http://localhost:3000/v1/auth/signup`
- **Authentication**: None

## Request Body

The request body should be sent as JSON. Below are the details of the required fields:

| Field         | Type   | Description                                    |
|---------------|--------|------------------------------------------------|
| `phone`       | string | The user's phone number, including country code. |
| `otp`         | string | The one-time password for verifying the user's phone number. |
| `email`       | string | The user's email address.                      |
| `first_name`  | string | The user's first name.                         |
| `last_name`   | string | The user's last name.                          |
| `date_of_birth` | string | The user's date of birth in DD/MM/YYYY format. |
| `password`    | string | The user's password.                           |
| `type`        | string | User type, e.g., "student".                    |
| `employee_id` | string | (Optional) The user's employee ID, if applicable. |
| `company`     | string | (Optional) The user's company name, if applicable. |
| `college`     | string | The user's college name.                       |
| `student_id`  | string | The user's student ID.                         |

### Example Request Body

```json
{
  "phone": "+918266187862",
  "otp": "123456",
  "email": "manasveer6@gmail.com",
  "first_name": "Manas",
  "last_name": "Veer",
  "date_of_birth": "24/02/2005",
  "password": "passwordgobrr",
  "type": "student",
  "employee_id": "",
  "company": "",
  "college": "AU",
  "student_id": "62"
}
```

## Response

Upon successful registration, the server will respond with appropriate status and the details of the created account. Be sure to handle possible errors, such as invalid input or server issues, as per your application's requirements.
