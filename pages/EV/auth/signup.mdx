# Signup API

## Endpoint

**POST** `http://localhost:3000/v1/auth/signup`

## Description

This endpoint allows users to sign up by providing their personal and authentication details.

## Request

### Headers

No authentication required.

### Body

The request body should be sent in JSON format with the following fields:

```json
{
  "phone": "+919812940706",
  "otp": "316892",
  "email": "mellob.mailer@gmail.com",
  "first_name": "Kartik",
  "last_name": "Deshmukh",
  "date_of_birth": "27/11/2004",
  "password": "password",
  "type": "student",
  "employee_id": "",
  "company": "",
  "college": "AU",
  "student_id": "34"
}
```

#### Fields

- `phone` (string): User's phone number, required.
- `otp` (string): One-time password for verification, required.
- `email` (string): User's email address, required.
- `first_name` (string): User's first name, required.
- `last_name` (string): User's last name, required.
- `date_of_birth` (string): User's date of birth in the format DD/MM/YYYY, required.
- `password` (string): User's password, required.
- `type` (string): Type of user, required. Possible values include `student` or other types depending on the system.
- `employee_id` (string): Employee ID, optional if user type is not an employee.
- `company` (string): Name of the company, optional if user type is not an employee.
- `college` (string): Name of the college, required if user type is student.
- `student_id` (string): Student ID, required if user type is student.

## Response

The server will respond with appropriate success or error messages based on the provided input.
