# Signup API

The Signup API allows users to create a new account by providing necessary personal details and account credentials.

## Endpoint

### POST

- **URL**: `http://localhost:3000/v1/auth/signup`
- **Body Format**: JSON
- **Authentication**: None required

## Request Body

The following JSON object should be sent in the body of the request:

```json
{
  "phone": "+918266187862",
  "otp": "123456",
  "email": "manasveer6@gmail.com",
  "first_name": "Manas",
  "last_name": "Veer",
  "date_of_birth": "24/02/2005",
  "password": "passwordgobrr",
  "type": "student",
  "employee_id": "",
  "company": "",
  "college": "AU",
  "student_id": "62"
}
```

### Fields

- **phone**: The user's phone number for verification.
- **otp**: One-time password for authentication.
- **email**: User's email address.
- **first_name**: The first name of the user.
- **last_name**: The last name of the user.
- **date_of_birth**: The user's date of birth in `DD/MM/YYYY` format.
- **password**: Password for the new account.
- **type**: User type, can be `student`.
- **employee_id**: Optional. The employee identification number, if applicable.
- **company**: Optional. The company the user is affiliated with, if applicable.
- **college**: The college the user is attending or affiliated with.
- **student_id**: The student identification number.

## Response

The response will confirm the successful creation of the account or provide error details if the signup fails. 

## Notes

- Ensure that the OTP is valid and matches the one sent to the phone number provided.
- Validate the password strength according to your application's password policy.
- Required fields must be filled out; optional fields can be left empty if not applicable.
