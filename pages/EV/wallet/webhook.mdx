# Webhook API Documentation

## Overview

This document describes the **Webhook API**. The Webhook API is used to interact with a wallet service via HTTP. It requires bearer token authentication and involves querying user details such as `uid`, `balance`, and a `webhook_key`.

## Endpoint

- **Type:** GET
- **URL:** `http://localhost:3000/v1/wallet/webhook`

### Query Parameters

| Name          | Type   | Description                     | Example                                       |
| ------------- | ------ | ------------------------------- | --------------------------------------------- |
| `uid`         | String | Unique user identifier          | `5geapgapxs`                                  |
| `balance`     | String | User's wallet balance           | `10000000000`                                 |
| `webhook_key` | String | Secret key for webhook access   | `coffeecodesEVBackendKarmafdmfbmd3brh4rfembvfadsadmbdbbvda` |

### Example URL

```plaintext
http://localhost:3000/v1/wallet/webhook?uid=5geapgapxs&balance=10000000000&webhook_key=coffeecodesEVBackendKarmafdmfbmd3brh4rfembvfadsadmbdbbvda
```

## Request Body

The GET request to this endpoint does not require a body.

## Summary

The Webhook API is intended to securely transmit user data, specifically the `uid`, `balance`, and `webhook_key`. Proper authorization using a bearer token is necessary for a successful API call. Ensure the query parameters are properly utilized to access the intended user data.
