# Webhook API Documentation

The Webhook API allows clients to interact with the system using HTTP methods. The current version supports a GET request to retrieve wallet details using specific query parameters. The API implements bearer token authentication for securing access.

## GET Request

**Endpoint:**

```
GET http://localhost:3000/v1/wallet/webhook
```

### Query Parameters

The following query parameters are required for the GET request.

- **uid**: The unique identifier for the user.
  - Example: `5geapgapxs`

- **balance**: The balance of the wallet.
  - Example: `10000000000`

- **webhook_key**: A key for accessing the webhook securely.
  - Example: `coffeecodesEVBackendKarmafdmfbmd3brh4rfembvfadsadmbdbbvda`

### Authentication

This API endpoint requires bearer token authentication. Pass the token in the HTTP header as shown below:

- **Authorization**: `Bearer <token>`

**Example Token:**

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJwaG9uZSI6Iis5MTk4MTI5NDA3MDYiLCJ1aWQiOiIwXzZpaGFjanJyIn0.Rf1ZUJpJdBBbep4cGCqwBQ2A8IyETp2UTrnecGHdi6I
```

### Example CURL Request

```bash
curl -X GET "http://localhost:3000/v1/wallet/webhook?uid=5geapgapxs&balance=10000000000&webhook_key=coffeecodesEVBackendKarmafdmfbmd3brh4rfembvfadsadmbdbbvda" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJwaG9uZSI6Iis5MTk4MTI5NDA3MDYiLCJ1aWQiOiIwXzZpaGFjanJyIn0.Rf1ZUJpJdBBbep4cGCqwBQ2A8IyETp2UTrnecGHdi6I"
```

This request will execute a GET operation on the webhook endpoint, returning wallet details based on the provided query parameters and valid authentication token.

Ensure that you replace the URL and tokens with the appropriate values when making actual requests.
