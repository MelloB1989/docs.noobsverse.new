# createSubscription API

The `createSubscription` API endpoint allows you to create a new subscription plan.

## Endpoint Details

- **Method:** POST
- **URL:** `http://localhost:3000/v1/subscriptions/create`
- **Authentication:** None required

## Headers

Include the following header for authorization purposes.

- **X-Karma-Admin-Auth:** `CoffeeCodesMelloBxAparnaEewqjbdkwqbkdjoehui3r7328grehjvdjvchjiuiudhsjkhkjsadab`

## Request Body

The request body must be in JSON format. Below is the structure you should follow when creating a subscription.

```json
[
  {
    "id": "sub_001",
    "monthly_fee": 29.99,
    "discount": 5.00,
    "name": "Standard Plan",
    "bike_id": "01rvo0_nbh",
    "type": "monthly",
    "security_deposit": 10
  },
  {
    "id": "sub_002",
    "monthly_fee": 39.99,
    "discount": 10.00,
    "name": "Premium Plan",
    "bike_id": "9tkdp5ra3p",
    "type": "monthly",
    "security_deposit": 10
  },
  {
    "id": "sub_003",
    "monthly_fee": 19.99,
    "discount": 0.00,
    "name": "Basic Plan",
    "bike_id": "2_vnd6qjv1",
    "type": "monthly",
    "security_deposit": 10

  {

    "monthly_fee": 49.99,
    "discount": 15.00,

    "bike_id": "vu1v4kb4s6",
    "type": "annual",
    "security_deposit": 10
  },
  {
    "id": "sub_005",
    "monthly_fee": 25.99,
    "discount": 5.00,
    "name": "Student Plan",
    "bike_id": "01rvo0_nbh",
    "type": "monthly",
    "security_deposit": 10
  }
]
```

Each object in the JSON array represents a subscription plan with the following fields:

- **id**: A unique identifier for the subscription plan.
- **monthly_fee**: The monthly fee for the subscription.
- **discount**: The discount applied to the subscription.
- **name**: The name of the subscription plan.
- **bike_id**: The identifier of the bike associated with the subscription.
- **type**: The type of subscription, e.g., "monthly" or "annual".
- **security_deposit**: The security deposit required for the subscription.

## Usage

To create a subscription, send a POST request to the specified URL with the correct headers and JSON-formatted body. The server will process the request and create the corresponding subscription plan.

## Response

On successful creation, the server will return a confirmation message with the details of the newly created subscription plan. If there are any errors, the server will respond with an appropriate error message.
