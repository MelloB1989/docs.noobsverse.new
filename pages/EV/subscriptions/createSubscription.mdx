# createSubscription API Documentation

## Overview

The `createSubscription` API is used to create new subscription plans in the system. This API supports multiple subscription options with different attributes for each plan.

## Endpoint

- **URL**: `http://localhost:3000/v1/subscriptions/create`
- **Method**: `POST`
- **Authentication**: None required

## Headers

- `X-Karma-Admin-Auth`: `CoffeeCodesMelloBxAparnaEewqjbdkwqbkdjoehui3r7328grehjvdjvchjiuiudhsjkhkjsadab`

## Request Body

### Content Type

- `application/json`

### Body Structure

The request body should contain an array of subscription objects. Each object must include the following fields:

- **id**: `string`
  - The unique identifier for the subscription.
  
- **monthly_fee**: `number`
  - The monthly fee associated with the subscription plan.
  
- **discount**: `number`
  - The discount amount applicable to the subscription.
  
- **name**: `string`
  - The name of the subscription plan.
  
- **bike_id**: `string`
  - The identifier for the bike associated with this subscription.
  
- **type**: `string`
  - The type of subscription, can be either `monthly` or `annual`.
  
- **security_deposit**: `number`
  - The security deposit amount required for the subscription.

## Response

The response from the server will contain the status of the subscription creation operation. Responses may vary depending on the server implementation and are not detailed here.

## Notes

- Ensure the `X-Karma-Admin-Auth` header value is correct to authorize the creation of subscriptions.
- The API currently does not require user authentication beyond the provided header.
- Check each field's data type to match the format specified above to avoid errors during the request.
