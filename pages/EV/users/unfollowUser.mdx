# unfollowUser API

The `unfollowUser` API allows authenticated users to unfollow a specified user. This is achieved via a `GET` request to the API endpoint.

## HTTP Request

- **Method**: `GET`
- **URL**: `http://localhost:3000/v1/user/unfollow/-sx7zvg7n1`
- **Body**: None

## Authorization

This API endpoint requires a Bearer token for authentication.

- **Authorization Type**: Bearer Token
- **Token Example**: 
  ```
  eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJwaG9uZSI6Iis5MTk4MTI5NDA3MDYiLCJ1aWQiOiI1Z2VhcGdhcHhzIn0.yNyKfoOQouzSNDwp_5rJ9CAEMVJo5uqueJHNpG-a3Aw
  ```

Make sure to replace the example token with a valid token obtained from your authentication system.

## Endpoint Details

### URL Parameters

- **User ID**: The URL segment `-sx7zvg7n1` is a placeholder for the user ID whom you want to unfollow. Replace it with the actual user ID.

### Example Request

```http
GET /v1/user/unfollow/[USER_ID] HTTP/1.1
Host: localhost:3000
Authorization: Bearer YOUR_ACCESS_TOKEN
```

Replace `[USER_ID]` with the actual user ID and `YOUR_ACCESS_TOKEN` with your actual Bearer token.

## Response

The response will indicate the success or failure of the unfollow operation. Ensure that the Bearer token used is valid and has the necessary permissions.

> **Note**: Ensure that the server is running and accessible at `localhost:3000` to utilize this API. Adjust the base URL accordingly if your server is hosted elsewhere.
