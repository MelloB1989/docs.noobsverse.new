## Description

The `unfollowUser` API endpoint allows a user to unfollow another user. This endpoint is part of the user management system and requires authentication via a bearer token.

## Endpoint Information

- **Type**: HTTP
- **Sequence**: 6

## HTTP Method

- **GET**

## Endpoint URL

```
http://localhost:3000/v1/user/unfollow/-sx7zvg7n1
```

- **Authorization**: `Bearer {token}`

## Response

The API will respond with appropriate status codes to indicate the result of the unfollow operation. For example:

- **200 OK**: The request was successful, and the user has been unfollowed.
- **401 Unauthorized**: Authentication failed, possibly due to an invalid or expired token.
- **404 Not Found**: The specified user to unfollow does not exist. 

Ensure to handle these response statuses when integrating this endpoint into your application.
